# Whetstone File Format Spec (v0.1)

Whetstone's entire state is plain markdown in one folder. Any tool that reads and writes
these formats is a Whetstone client — the agent skill, the hub page, a phone text editor,
or something you build yourself. **The files are the API.**

```
<whetstone folder>/
  whetstone.json      # config: grading tier, sync note, created date
  PENDING.md          # today's worksheet — always exists
  profile.md          # learner profile: recurring misconception patterns
  decks/
    <source-slug>.md  # one deck per consumed source
```

## Deck file

YAML frontmatter, a short header paragraph (source, scope, notable results), then items:

```markdown
---
title: "Serrano 2024: PPO for Training LLMs"
source: "https://www.youtube.com/watch?v=..."
created: 2026-07-12
tags: [rl, ppo]
---

One-paragraph header: what the source covers, when ingested, current weak spots.

### Q1
- prompt: Teach-back — explain the surrogate objective and why it uses the probability ratio.
- answer: The ratio is an importance-sampling correction for data collected by the old policy...
- stage: 0
- due: 2026-07-13
- history: 2026-07-12 fail (confused ratio with momentum)
- status: active
```

Field rules:
- `prompt` / `answer`: single line each (use "; " for structure). MCQ prompts start with
  `MCQ —` and embed options as `(a) ... (b) ... (c) ... (d) ...`; the answer starts with the
  correct letter, e.g. `(b). Because ...`.
- `stage`: integer 0–4, index into the interval ladder `[1, 3, 7, 16, 35]` days.
- `due`: `YYYY-MM-DD`. Item is due when `due <= today` and `status: active`.
- `history`: comma-separated, append-only: `YYYY-MM-DD pass|partial|fail (context)`.
- `status`: `active` | `retired`. Retired items are never scheduled again.

## PENDING.md — the worksheet

Always exists; regenerated by the skill after every grading pass. Contains exactly the
currently due items (max 8), formatted so a human can answer inline in any text editor:

```markdown
# Whetstone — Pending Worksheet
Generated: 2026-07-13 · Answer inline, then run /whetstone to grade & reschedule.

## 1. [serrano-2024-ppo-video / Q4] MCQ
Advantage is positive and the ratio hits 1+ε. What does the clipped objective do?
- [ ] (a) scales the gradient by ε
- [ ] (b) gradient becomes zero — no incentive to push further
- [ ] (c) reverses the gradient
- [ ] (d) resets the ratio to 1

## 2. [serrano-2024-ppo-video / Q3] teach-back
Explain the surrogate objective. Why the probability ratio times the advantage?

> answer:

```

Rules:
- Item header: `## <n>. [<deck-slug> / <item-id>] <MCQ|teach-back|recall>`.
- MCQ: user checks exactly one box (`- [x]`).
- Text items: user writes below `> answer:` (any number of lines until the next `##`).
- Clients append answers only; the skill is the only writer that regenerates the file.

## profile.md — learner profile

Append-only list of *how* the user misunderstands, not what they got wrong:

```markdown
## Misconception patterns
- 2026-07-12 — Conflates an objective's value with its gradient (PPO clipping). Watch for
  gradient-flow reasoning; target with items that ask "what is the derivative doing here?"
```

Ingestion consults this file to write items that target known patterns.

## whetstone.json

```json
{ "grading": "deferred", "sync": "icloud", "created": "2026-07-12", "version": "0.1" }
```

`grading`: `deferred` (default — hub queues answers, skill grades; zero API cost) or `live`
(hub calls an LLM with the user's key).

## Scheduling contract (all clients)

pass → stage+1 (retire on pass at stage 4) · partial → stage unchanged · fail → stage−1 (min 0)
· then `due = today + [1,3,7,16,35][stage]` · append history · never delete history.
